= BlueXP disaster recovery现已更名为NetApp灾难恢复
:allow-uri-read: 




=== BlueXP disaster recovery现已更名为NetApp灾难恢复

BlueXP disaster recovery已重命名为NetApp灾难恢复。



=== BlueXP现在是NetApp控制台

NetApp控制台建立在增强和重组的BlueXP基础之上，可在企业级内部和云环境中集中管理NetApp存储和NetApp数据服务，提供实时洞察、更快的工作流程和简化的管理，并且高度安全且合规。

有关更改的详细信息，请参阅link:https://docs.netapp.com/us-en/bluexp-relnotes/index.html["NetApp控制台发行说明"]。



=== 其他更新

* Amazon Elastic VMware Service (EVS) 与Amazon FSx for NetApp ONTAP的支持已公开预览。随着此版本的发布，它现已普遍可用。有关详细信息，请参阅link:../reference/evs-deploy-guide-introduction.html["使用 Amazon Elastic VMware Service 和Amazon FSx for NetApp ONTAP实现NetApp灾难恢复"]。
* 存储发现改进，包括减少本地部署的发现时间
* 身份和访问管理 (IAM) 支持，包括基于角色的访问控制 (RBAC) 和增强的用户权限
* 对 Azure VMware 解决方案和Cloud Volumes ONTAP 的私人预览支持。有了此支持，您现在可以使用Cloud Volumes ONTAP存储配置从本地到 Azure VMware 解决方案的灾难恢复保护。
